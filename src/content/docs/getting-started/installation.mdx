To run Clips locally, install the pinix daemon.

## System Requirements

| | Supported |
|---|---|
| **Operating system** | macOS, Linux |
| **Architecture** | arm64 (Apple Silicon, ARM servers), amd64 (Intel/AMD) |

<Aside>
  Windows is not supported yet. You can use [Docker](#docker) or WSL2.
</Aside>

## Installation

<Steps>

1. **Install with one command**

   ```bash
   curl -fsSL dl.pinixai.com/install.sh | sh
   ```

   This installs automatically:

   | Component | Purpose |
   |-----------|---------|
   | `pinix` | CLI + daemon (single binary) |
   | Node.js | Runtime dependency (installed automatically through fnm if missing) |
   | Bun | Clip runtime, used to run TypeScript Clips |
   | bb-browser | Browser automation Edge Clip |

   Chrome is downloaded automatically the first time the daemon starts.

2. **Start the daemon**

   ```bash
   pinix start
   ```

   The daemon starts on port :9000 and automatically starts Chrome and bb-browser.

3. **Log in**

   ```bash
   pinix login
   ```

   This opens your browser for authorization. After login, the daemon automatically connects to the Cloud Hub, so your local Clips can also be accessed from the cloud.

   <Aside type="tip">
     You can also log in with a token: `pinix login --token pnx_...` (get the token from [pinixai.com/settings](https://pinixai.com/settings)).
   </Aside>

4. **Open the Console**

   ```bash
   open http://localhost:9000
   ```

   The local Console has the same features as the cloud Console, but all Clips run on your machine.

</Steps>

## Try It

After startup, the bb-browser platform Clips are already registered:

```bash
# 查看可用的 Clip
pinix hub list

# 搜索 HackerNews
pinix invoke hackernews top

# 搜索 Google
pinix invoke google search "pinix ai agent"

# 安装一个 Clip
pinix hub add @pinix/todo

# 使用它
pinix invoke todo list
```

## Common Commands

| Command | Description |
|---------|-------------|
| `pinix start` | Start the daemon |
| `pinix stop` | Stop the daemon |
| `pinix hub list` | View installed Clips |
| `pinix hub add @scope/name` | Install a Clip |
| `pinix hub remove <alias>` | Uninstall a Clip |
| `pinix invoke <alias> <command>` | Invoke a Clip |
| `pinix upgrade` | Upgrade Pinix |
| `pinix whoami` | Check login status |

## Docker

If you do not want to install anything on the host machine, use Docker:

```bash
docker run -d --shm-size=2g lueco/pinix <your-hub-token>
```

The image includes pinix, Bun, Node.js, bb-browser, Chrome, and Xvfb. After startup, it logs in automatically and runs the daemon.

Get the Hub token from [pinixai.com/settings](https://pinixai.com/settings).

## Build from Source

Requires Go 1.25+:

```bash
git clone https://github.com/epiral/pinix.git
cd pinix
make build
```

You also need to install Bun and bb-browser separately:

```bash
curl -fsSL https://bun.sh/install | bash
npm install -g bb-browser
```

## Upgrade

```bash
pinix upgrade
```

This automatically downloads the latest binary and updates bb-browser.
