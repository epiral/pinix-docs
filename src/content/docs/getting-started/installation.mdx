Pinix has two installation modes, depending on your use case:

| Installation mode | Best for | What it installs |
|---------|-------|---------|
| **Full installation** | People who want to run Clips on this machine | daemon + CLI + browser engine + runtime |
| **CLI-only** | People who want to invoke other people's Clips remotely | Only the `pinix` command-line tool |

Typical setup: one server uses the full installation to run the daemon and Clips, while the rest of the team installs CLI-only on their own machines for remote invocation. Or, if you are already using the [pinixai.com Cloud Agent](/getting-started/), you can install only the CLI to invoke Clips from your terminal too.

## System Requirements

| | macOS | Linux | Windows |
|---|---|---|---|
| **Full installation** | arm64 / amd64 | arm64 / amd64 | amd64 |
| **CLI-only** | arm64 / amd64 | arm64 / amd64 | amd64 |

## Full Installation

Install the daemon and all dependencies to run Clips on this machine.

<Tabs>
  <TabItem label="macOS / Linux">

```bash
curl -fsSL dl.pinixai.com/install.sh | sh
```

  </TabItem>
  <TabItem label="Windows (PowerShell)">

```powershell
irm dl.pinixai.com/install.ps1 | iex
```

  </TabItem>
  <TabItem label="Docker">

```bash
docker run -d --shm-size=2g lueco/pinix <your-hub-token>
```

Get the Hub token from [pinixai.com/settings](https://pinixai.com/settings). The image includes every component.

  </TabItem>
</Tabs>

Components installed automatically:

| Component | Purpose |
|------|------|
| `pinix` | CLI + daemon (single binary) |
| Node.js | Runtime dependency (installed automatically if missing) |
| Bun | Clip runtime for running TypeScript Clips |
| bb-browser | Browser automation Edge Clip |

Chrome is downloaded automatically the first time the daemon starts.

### Start

<Steps>

1. **Start the daemon**

   ```bash
   pinix start
   ```

   The daemon starts on port :9000 and automatically launches Chrome and bb-browser.

2. **Log in**

   ```bash
   # 浏览器授权（会自动打开浏览器）
   pinix login

   # 或者用 token 登录（从 pinixai.com/settings 获取 token）
   pinix login --token pnx_...
   ```

   After login, the daemon automatically connects to the Cloud Hub. Your local Clips can be accessed from the cloud, and you can also use Clips shared by others.

3. **Start using Pinix**

   ```bash
   # 查看可用的 Clip
   pinix hub list

   # bb-browser 的平台 Clip 已自动注册
   pinix invoke hackernews top
   pinix invoke google search "pinix ai agent"

   # 安装更多 Clip
   pinix hub add @pinix/todo
   pinix invoke todo list

   # 打开本地 Console
   open http://localhost:9000
   ```

</Steps>

## CLI-only Installation

Install only the `pinix` command-line tool, without starting the daemon or installing runtimes. Use it to invoke Clips through the Cloud Hub or a remote daemon.

<Tabs>
  <TabItem label="macOS / Linux">

```bash
curl -fsSL dl.pinixai.com/install.sh | sh -s -- --cli
```

  </TabItem>
  <TabItem label="Windows (PowerShell)">

```powershell
irm dl.pinixai.com/install.ps1 | iex  # 选择 CLI-only 模式
```

  </TabItem>
</Tabs>

### Log In

After installation, you need to log in. There are two ways:

<Tabs>
  <TabItem label="Browser Authorization">

```bash
pinix login
```

This prints a link and automatically opens your browser. Confirm authorization on the web page to finish.

  </TabItem>
  <TabItem label="Token Login">

```bash
pinix login --token pnx_...
```

Get the token from the **Hub Token** section at [pinixai.com/settings](https://pinixai.com/settings). This is suitable for browserless environments such as servers and CI/CD.

  </TabItem>
</Tabs>

After login, the CLI automatically connects to the Cloud Hub. You do not need to manually specify `--server`; use it directly:

```bash
# 查看所有可用的 Clip（你自己的 + 别人共享的）
pinix hub list

# 调用 Clip
pinix invoke todo list
pinix invoke hackernews top
```

### Directly Connect to a LAN Daemon

If you do not want to go through the Cloud Hub, you can also connect directly to another machine's daemon on the local network:

```bash
pinix --server http://192.168.1.100:9000 hub list
pinix --server http://192.168.1.100:9000 invoke todo list
```

<Aside type="tip">
  CLI-only is a good fit for these scenarios:
  - A team shares one daemon server, and everyone else only needs CLI invocation
  - Invoking Clips in CI/CD (log in with a token)
  - Using Clips remotely from lightweight devices or Windows
  - You only want to use Clips from the terminal and do not need to run a local daemon
</Aside>

## Common Commands

| Command | Description | Requires daemon |
|------|------|:-----------:|
| `pinix start` | Start the daemon | — |
| `pinix stop` | Stop the daemon | — |
| `pinix login` | Log in | No |
| `pinix whoami` | Check login status | No |
| `pinix hub list` | View the Clip list | Yes |
| `pinix hub add @scope/name` | Install a Clip | Yes |
| `pinix hub remove <alias>` | Uninstall a Clip | Yes |
| `pinix hub update [alias]` | Update Clips to the latest version | Yes |
| `pinix hub info <alias>` | View Clip commands and parameters | Yes |
| `pinix invoke <alias> <command>` | Invoke a Clip | Yes |
| `pinix registry search <query>` | Search the Registry | No |
| `pinix upgrade` | Upgrade Pinix itself | No |

## Build from Source

Requires Go 1.25+:

```bash
git clone https://github.com/epiral/pinix.git
cd pinix
make build
```

You also need to install Bun and bb-browser separately:

```bash
curl -fsSL https://bun.sh/install | bash
npm install -g bb-browser
```

## Update Clips

```bash
# 更新单个 Clip 到最新版
pinix hub update todo

# 更新所有 Registry 安装的 Clip
pinix hub update --all

# 更新到指定版本
pinix hub update todo --version 1.2.0
```

<Aside>
  Only Clips installed from the Registry (`@scope/name`) can be updated. Locally installed and GitHub-installed Clips need to be added again with `hub add`.
</Aside>

## Upgrade Pinix

```bash
pinix upgrade
```

This upgrades the Pinix binary itself and updates bb-browser. It does not affect installed Clips.
