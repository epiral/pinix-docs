Pinix exposes all your installed Clips as MCP (Model Context Protocol) Tools. This means you can use them in Claude Desktop, Cursor, and any other MCP-compatible client.

## Two MCP modes

### Hub MCP (recommended)

Exposes **all Clips** on your Hub as MCP tools through three meta-tools:

| Meta-tool | Description |
|-----------|-------------|
| `pinix_list` | List all available Clips and their commands |
| `pinix_invoke` | Invoke any Clip command |
| `pinix_invoke_stream` | Invoke with streaming output |

The Agent discovers available Clips dynamically — when you install a new Clip, it's immediately available as an MCP tool without restarting.

### Single Clip MCP

Runs a specific Clip as a standalone MCP server. Each Clip command becomes a separate MCP tool. Useful when you want to expose just one Clip.

## Configuration

<Tabs>
  <TabItem label="Claude Desktop">

Edit `~/Library/Application Support/Claude/claude_desktop_config.json`:

**Hub MCP (all Clips):**
```json
{
  "mcpServers": {
    "pinix": {
      "command": "pinix",
      "args": ["mcp"]
    }
  }
}
```

**Single Clip MCP (one Clip):**
```json
{
  "mcpServers": {
    "todo": {
      "command": "pinix",
      "args": ["mcp", "--clip", "todo"]
    }
  }
}
```

Restart Claude Desktop after editing the config.

  </TabItem>
  <TabItem label="Cursor">

Add to Cursor's MCP settings (Settings > MCP):

**Hub MCP (all Clips):**
```json
{
  "mcpServers": {
    "pinix": {
      "command": "pinix",
      "args": ["mcp"]
    }
  }
}
```

  </TabItem>

  }

## How it works

When an MCP client connects to `pinix mcp`:

1. The MCP server connects to your local Hub (port 9000)
2. It queries all registered Clips and their commands
3. It exposes them as MCP tools (Hub mode) or individual tools (Single Clip mode)
4. When the Agent calls a tool, the MCP server invokes the corresponding Clip through the Hub

```
Claude Desktop → MCP Protocol → pinix mcp → Hub → Clip
```

<Aside>
  You need `pinix start` running for MCP to work — the MCP server connects to your local Hub. If the daemon isn't running, the MCP server will fail to connect.
</Aside>

## Verifying the setup

After configuring MCP, you can verify it works:

1. Start the daemon: `pinix start`
2. Open Claude Desktop (or your MCP client)
3. Ask: *"List my Pinix clips"* — the Agent should call `pinix_list` and show your installed Clips
4. Ask: *"Search HackerNews for top stories"* — the Agent should invoke the hackernews Clip
