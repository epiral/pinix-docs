Pinix 通过两个概念，把任何设备变成 AI 可访问的能力来源：**Provider** 和 **Edge Clip**。

## 什么是 Provider？

Provider 是一种连接 —— 一个服务，负责将 Clip 注册到 Hub，并把调用转发给它们。Provider 协议（`ProviderStream`）负责：

- **注册**：告诉 Hub 哪些 Clip 可用
- **调用转发**：将 invoke 请求送到正确的 Clip
- **心跳**：保持连接存活

Provider 有两类：

| Provider type | What it manages | Example |
|---|---|---|
| **Runtime** | SDK Clip（Bun/TS 进程） | `pinixd` 内置的 Runtime |
| **Edge Clip Provider** | 自身 —— Provider *就是* Clip | bb-browser, clip-dock-macos |

Runtime 是一种特殊的 Provider，用来管理多个 SDK Clip。Edge Clip Provider 只管理它自己。

## 什么是 Edge Clip？

**Edge Clip** 直接绑定硬件或 OS API。它自己实现 Provider 协议，并作为独立服务连接 Hub。

```
┌────────────────────────────┐
│      bb-browser            │
│  (Edge Clip Provider)      │
│                            │
│  ├─ Controls Chrome        │
│  ├─ Site adapters          │     ProviderStream
│  │  ├─ twitter             │ ◄──────────────────► Hub
│  │  ├─ hackernews          │
│  │  ├─ google              │
│  │  └─ 100+ more           │
│  └─ Browser automation     │
└────────────────────────────┘
```

### 为什么需要 Edge Clip？

有些能力无法打包成 TypeScript 模块：

- **浏览器自动化**需要控制真实的 Chrome 实例
- **macOS 集成**需要原生 API（截图、剪贴板、通知）
- **IoT 设备**需要直接访问硬件
- **移动 App** 需要平台特定框架

Edge Clip 通过在设备上原生运行，并通过 Provider 协议暴露结构化命令来解决这个问题。对 Hub（以及 Agent）来说，它们看起来和任何其他 Clip 完全一样。

## BB-Browser：参考 Edge Clip

[BB-Browser](https://github.com/epiral/bb-browser) 是 Pinix 的浏览器自动化 Edge Clip。它把你的浏览器变成 API：

- **36 个平台**：Twitter、Google、HackerNews、Reddit、Wikipedia、小红书等
- **103 个命令**：search、read、post、monitor、extract
- **站点适配器**：针对每个平台做结构化提取，而不是通用爬取

BB-Browser 会由 `pinix start` 自动启动。它会启动一个 headed Chrome 实例，并将平台命令注册到 Hub。

<Aside>
  Chrome 以 **headed mode**（不是 headless）运行，以避免反自动化检测。在 macOS 上它会离屏渲染；在 Linux 上，pinixd 会自动启动 Xvfb。
</Aside>

## clip-dock：设备能力包

`clip-dock` 是一种设备侧产品概念：将设备的所有能力作为 Edge Clip 暴露出来。

在 Mac 上安装 `clip-dock-macos` 后，Agent 可以立即：
- 截图
- 读取和写入剪贴板
- 发送通知
- 访问文件系统
- 控制系统设置

目标是：**一次安装 = 任何 Agent 都可使用全部设备能力**。

## 设备抽象模式

Edge Clip 暴露原始设备能力。常规 SDK Clip 可以在其上构建抽象：

```
Raw capability (Edge Clip):
  browser.navigate → opens a URL in Chrome

Abstraction (SDK Clip):
  twitter.search → uses browser.navigate + site adapter
                    to search Twitter and return structured results
```

这种分层方式意味着：
1. Edge Clip 开发者专注于可靠的设备访问
2. SDK Clip 开发者专注于领域逻辑
3. Agent 获得清晰的高层命令
