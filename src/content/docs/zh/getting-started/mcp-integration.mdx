Pinix 会将你安装的所有 Clips 暴露为 MCP（Model Context Protocol）Tools。这意味着你可以在 Claude Desktop、Cursor 以及任何兼容 MCP 的客户端中使用它们。

## 两种 MCP 模式

### Hub MCP（推荐）

通过三个 meta-tools，将你 Hub 上的 **所有 Clips** 暴露为 MCP tools：

| Meta-tool | 描述 |
|-----------|-------------|
| `pinix_list` | 列出所有可用 Clips 及其命令 |
| `pinix_invoke` | 调用任意 Clip 命令 |
| `pinix_invoke_stream` | 以流式输出调用 |

Agent 会动态发现可用 Clips —— 当你安装新的 Clip 时，它会立即作为 MCP tool 可用，无需重启。

### Single Clip MCP

将某个特定 Clip 作为独立 MCP server 运行。每个 Clip 命令都会成为一个单独的 MCP tool。适合只想暴露一个 Clip 的场景。

## 配置

<Tabs>
  <TabItem label="Claude Desktop">

编辑 `~/Library/Application Support/Claude/claude_desktop_config.json`：

**Hub MCP（所有 Clips）：**
```json
{
  "mcpServers": {
    "pinix": {
      "command": "pinix",
      "args": ["mcp"]
    }
  }
}
```

**Single Clip MCP（一个 Clip）：**
```json
{
  "mcpServers": {
    "todo": {
      "command": "pinix",
      "args": ["mcp", "--clip", "todo"]
    }
  }
}
```

编辑配置后重启 Claude Desktop。

  </TabItem>
  <TabItem label="Cursor">

添加到 Cursor 的 MCP 设置中（Settings > MCP）：

**Hub MCP（所有 Clips）：**
```json
{
  "mcpServers": {
    "pinix": {
      "command": "pinix",
      "args": ["mcp"]
    }
  }
}
```

  </TabItem>
  <TabItem label="OpenCode">

添加到 `opencode.json`：

```json
{
  "mcp": {
    "pinix": {
      "type": "stdio",
      "command": "pinix",
      "args": ["mcp"]
    }
  }
}
```

  </TabItem>
</Tabs>

## 工作原理

当 MCP 客户端连接到 `pinix mcp` 时：

1. MCP server 连接到你的本地 Hub（9000 端口）
2. 它查询所有已注册的 Clips 及其命令
3. 它将它们暴露为 MCP tools（Hub 模式）或独立 tools（Single Clip 模式）
4. 当 Agent 调用一个 tool 时，MCP server 会通过 Hub 调用对应的 Clip

```
Claude Desktop → MCP Protocol → pinix mcp → Hub → Clip
```

<Aside>
  MCP 需要先运行 `pinix start` —— MCP server 会连接到你的本地 Hub。如果 daemon 没有运行，MCP server 将无法连接。
</Aside>

## 验证设置

配置 MCP 后，你可以验证它是否工作：

1. 启动 daemon：`pinix start`
2. 打开 Claude Desktop（或你的 MCP 客户端）
3. 询问：*"List my Pinix clips"* —— Agent 应该会调用 `pinix_list` 并显示你安装的 Clips
4. 询问：*"Search HackerNews for top stories"* —— Agent 应该会调用 hackernews Clip
