Pinix 有两种安装方式，取决于你的使用场景：

| 安装方式 | 适合谁 | 安装什么 |
|---------|-------|---------|
| **完整安装** | 要在这台机器上运行 Clip 的人 | daemon + CLI + 浏览器引擎 + 运行时 |
| **CLI-only** | 要远程调用别人 Clip 的人 | 只装 `pinix` 命令行 |

典型场景：一台服务器做完整安装跑 daemon 和 Clip，团队其他人在自己电脑上装 CLI-only 远程调用。或者你已经在用 [pinixai.com 云端 Agent](/zh/getting-started/)，只想在终端里也能调用 Clip。

## 系统要求

| | macOS | Linux | Windows |
|---|---|---|---|
| **完整安装** | arm64 / amd64 | arm64 / amd64 | amd64 |
| **CLI-only** | arm64 / amd64 | arm64 / amd64 | amd64 |

## 完整安装

安装 daemon + 所有依赖，在本机运行 Clip。

<Tabs>
  <TabItem label="macOS / Linux">

```bash
curl -fsSL dl.pinixai.com/install.sh | sh
```

  </TabItem>
  <TabItem label="Windows (PowerShell)">

```powershell
irm dl.pinixai.com/install.ps1 | iex
```

  </TabItem>
  <TabItem label="Docker">

```bash
docker run -d --shm-size=2g lueco/pinix <your-hub-token>
```

Hub token 从 [pinixai.com/settings](https://pinixai.com/settings) 获取。镜像包含全部组件。

  </TabItem>
</Tabs>

自动安装的组件：

| 组件 | 用途 |
|------|------|
| `pinix` | CLI + daemon（单一二进制） |
| Node.js | 运行时依赖（没有会自动安装） |
| Bun | Clip 运行时——运行 TypeScript Clip |
| bb-browser | 浏览器自动化 Edge Clip |

Chrome 会在首次启动 daemon 时自动下载。

### 启动

<Steps>

1. **启动 daemon**

   ```bash
   pinix start
   ```

   daemon 启动在 :9000 端口，同时自动拉起 Chrome 和 bb-browser。

2. **登录**

   ```bash
   # 浏览器授权（会自动打开浏览器）
   pinix login

   # 或者用 token 登录（从 pinixai.com/settings 获取 token）
   pinix login --token pnx_...
   ```

   登录后，daemon 自动连接 Cloud Hub——你的本地 Clip 从云端也能访问，你也能用到别人共享的 Clip。

3. **开始使用**

   ```bash
   # 查看可用的 Clip
   pinix hub list

   # bb-browser 的平台 Clip 已自动注册
   pinix invoke hackernews top
   pinix invoke google search "pinix ai agent"

   # 安装更多 Clip
   pinix hub add @pinix/todo
   pinix invoke todo list

   # 打开本地 Console
   open http://localhost:9000
   ```

</Steps>

## CLI-only 安装

只安装 `pinix` 命令行工具，不启动 daemon，不装运行时。通过 Cloud Hub 或远程 daemon 调用 Clip。

<Tabs>
  <TabItem label="macOS / Linux">

```bash
curl -fsSL dl.pinixai.com/install.sh | sh -s -- --cli
```

  </TabItem>
  <TabItem label="Windows (PowerShell)">

```powershell
irm dl.pinixai.com/install.ps1 | iex  # 选择 CLI-only 模式
```

  </TabItem>
</Tabs>

### 登录

安装后需要登录，有两种方式：

<Tabs>
  <TabItem label="浏览器授权">

```bash
pinix login
```

会打印一个链接并自动打开浏览器，在网页上确认授权即可。

  </TabItem>
  <TabItem label="Token 登录">

```bash
pinix login --token pnx_...
```

Token 从 [pinixai.com/settings](https://pinixai.com/settings) 的 **Hub Token** 部分获取。适合无浏览器环境（服务器、CI/CD）。

  </TabItem>
</Tabs>

登录后，CLI 自动连接 Cloud Hub。不需要手动指定 `--server`，直接使用：

```bash
# 查看所有可用的 Clip（你自己的 + 别人共享的）
pinix hub list

# 调用 Clip
pinix invoke todo list
pinix invoke hackernews top
```

### 直连局域网 daemon

如果不走 Cloud Hub，也可以直接连接局域网内另一台机器的 daemon：

```bash
pinix --server http://192.168.1.100:9000 hub list
pinix --server http://192.168.1.100:9000 invoke todo list
```

<Aside type="tip">
  CLI-only 适合这些场景：
  - 团队共享一台 daemon 服务器，其他人只需要 CLI 调用
  - 在 CI/CD 中调用 Clip（用 token 登录）
  - 在轻量设备或 Windows 上远程使用 Clip
  - 只想在终端里用 Clip，不需要本地跑 daemon
</Aside>

## 常用命令

| 命令 | 说明 | 需要 daemon |
|------|------|:-----------:|
| `pinix start` | 启动 daemon | — |
| `pinix stop` | 停止 daemon | — |
| `pinix login` | 登录 | 否 |
| `pinix whoami` | 查看登录状态 | 否 |
| `pinix hub list` | 查看 Clip 列表 | 是 |
| `pinix hub add @scope/name` | 安装 Clip | 是 |
| `pinix hub remove <alias>` | 卸载 Clip | 是 |
| `pinix invoke <alias> <command>` | 调用 Clip | 是 |
| `pinix upgrade` | 升级 Pinix | 否 |

## 从源码构建

需要 Go 1.25+：

```bash
git clone https://github.com/epiral/pinix.git
cd pinix
make build
```

还需要单独安装 Bun 和 bb-browser：

```bash
curl -fsSL https://bun.sh/install | bash
npm install -g bb-browser
```

## 升级

```bash
pinix upgrade
```

自动下载最新二进制并更新 bb-browser。
