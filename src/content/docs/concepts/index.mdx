Pinix lets AI Agents use any capability on any device. Instead of building complex tool pipelines for each Agent, you install **Clips** — self-contained capability units that any Agent can discover and invoke through a **Hub**.

Understand these five concepts and you understand Pinix.

## How it fits together

```
User ──► Agent ──► Hub ──────► Clip (SDK)
                    │             ↑
                    │          Runtime (manages lifecycle)
                    │
                    └────────► Clip (Edge)
                                 ↑
                              Provider Protocol
                              (bb-browser, clip-dock, ...)
```

The **Agent** understands what the user wants. The **Hub** routes the request to the right **Clip**. The Clip executes the actual work — whether that's searching Twitter, managing a to-do list, or controlling a browser.

**SDK Clips** are TypeScript packages managed by the Runtime. **Edge Clips** bind directly to hardware or OS APIs through the Provider protocol.
